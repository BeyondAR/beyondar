15 ## Adding BeyondAR in to your project
16 
17 Just download the latest version of the framework [here](android/libs) and add the needed jar files in to you libs folder.
18 
19 * beyondar-v#.jar: The basic lib to be able to run the framework
20 * beyondar-googlemap-module-v#.jar: Module to use GoogleMaps with your `World `

22 ##How to build your first app
23 
24 To be able to run BeyondAR we need to add the following lines on the AndroidManifest.xml
25 
26 ~~~xml
27 <!-- Minimum permissions for Beyondar -->
28 <uses-permission android:name="android.permission.CAMERA" />
29 
30 <!-- For beyondar this is not mandatory unless you want to load something from Internet (for instance images) -->
31 <uses-permission android:name="android.permission.INTERNET" />
32 
33 <!-- BeyondAR needs the following features-->
34 <uses-feature android:name="android.hardware.camera" />
35 <uses-feature android:name="android.hardware.camera.autofocus" />
36 <uses-feature android:name="android.hardware.sensor.accelerometer" />
37 <uses-feature android:name="android.hardware.sensor.compass" />
38 ~~~

172 ## Adding GoogleMaps module
173 BeyondAR Framework uses modules to be able to add multiple features to the world engine. Google Maps Module is one example ([jar](https://github.com/BeyondAR/beyondar/tree/master/android/libs/modules) & [src](android/BeyondAR_Framework/src-googlemaps)).
174 
175 To draw the all the `World ` elements in the Google Map framework we just need a few lines of code:
176 
177 ~~~java
178 // ...
179 private GoogleMap mMap;
180 private GoogleMapWorldModule mGoogleMapModule;
181 // ...
182 
183 @Override
184 protected void onCreate(Bundle savedInstanceState) {
185  // ...
186  mMap = ((SupportMapFragment) getSupportFragmentManager().findFragmentById(R.id.map)).getMap();
187 
188  // We create the world...
189  mWorld = new World(this);
190  // As we want to use GoogleMaps, we are going to create the module and
191  // attach it to the World
192  mGoogleMapModule = new GoogleMapWorldModule(context);
193  // Then we need to set the map in to the GoogleMapModule
194  mGoogleMapModule.setGoogleMap(mMap);
195  // Now that we have the module created let's add it in to our world
196  mWorld.addModule(mGoogleMapModule);
197 
198  // Now we fill the world
199  // ...
200 }
201 ~~~
202 The `GoogleMapWorldModule ` will take care of drawing all the `GeoObjects ` in the `GoogleMap ` object. So we also can add the a listener to the map to get notify when a `Marker ` is click and then we can check which `GeoObject ` is the owner of that `Marker `:
203 
204 ~~~java
205 @Override
206 protected void onCreate(Bundle savedInstanceState) {
207  // ...
208  mMap.setOnMarkerClickListener(this);
209  // ...
210 }
211 
212 @Override
213 public boolean onMarkerClick(Marker marker) {
214  // To get the GeoObject that owns the marker we use the following
215  // method:
216  GeoObject geoObject = mGoogleMapModule.getGeoObjectOwner(marker);
217  if (geoObject != null) {
218  Toast.makeText(this, "Click on a marker owned by a GeoOject with the name: " + geoObject.getName(), Toast.LENGTH_SHORT).show();
219  }
220  return false;
221 }
222 ~~~

261 }

265 ## Creating your own module
266 
267 BeyondAR architecture allows you to create your own modules that can be attached to the framework. For instance to have a better access to the `World ` object in order to perform other extra task like using Google Maps, or a radar view.
268 
269 The first thing that we need to do is understand what do we need to implement in order to create our module. The interfaces are located in `com.beyondar.android.module `:
270 
271 * `WorldModule `: This interface will allow your module to be notified when some events occur, like when the position has changed, a new object has been added/removed, all the `World ` is cleaned, etc. ([here](android/BeyondAR_Framework/src/com/beyondar/android/module/WorldModule.java) you will find the code).
272 * `BeyondarObjectModule `: This interface allows your module to get notified when there are changes in a specific `BeyondarObject ` ([here](android/BeyondAR_Framework/src/com/beyondar/android/module/BeyondarObjectModule.java) you will find the code).
273 * `GeoObjectModule `: This interface extends `BeyondarObjectModule ` and it have some extra code to make easier the control of the geo position of this kind of objects ([here](android/BeyondAR_Framework/src/com/beyondar/android/module/BeyondarObjectModule.java) you will find the code).
274 
275 One of the main goals of `WorldModule ` is to add `BeyondarObjectModule `/`GeoObjectModule ` to all the `BeyondarObject `/`GeoObject ` in the `World ` object. To do that make sure to add you own module in when `setup(World world) ` is called and when a new `BeyondarObject ` is added:
276 
277 ~~~java
278 @Override
279 public void onBeyondarObjectAdded(BeyondarObject beyondarObject, BeyondarObjectList beyondarObjectList) {
280  if (beyondarObject instanceof GeoObject) { // Check if it is a GeoObject
281  // We need to check if there is any of our own module already attached
282  if (!beyondarObject.containsAnyModule(RadarPointModule.class)) {
283  // Then we just create it and add it
284  RadarPointModule module = new RadarPointModule(this, beyondarObject);
285  beyondarObject.addModule(module);
286  }
287  }
288 }
289 ~~~
290 
291 Once we have created the module we need to add it to the `World ` class, for that we just use the method `myWorld.addModule(myModule) `.
292 
293 ~~~java
294 World myWorld = new World(context);
295 // add the module
296 myWorld.addModule(myModule);
297 ~~~
